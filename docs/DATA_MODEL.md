# Data Model

## Overview

The database is Neon PostgreSQL, accessed via Drizzle ORM with the `@neondatabase/serverless` HTTP driver. Schema is defined in `lib/schema.ts`. The connection is initialized lazily in `lib/db.ts`.

## Schema

### Better Auth Tables

These are managed by Better Auth and should not be modified directly. They follow Better Auth's expected column names.

| `email` | text (unique) | From Vercel profile |
| `emailVerified` | boolean | |
| `image` | text | Avatar URL |
| `createdAt` | timestamp | |
| `updatedAt` | timestamp | |

**`session`** - auth sessions (not agent sessions)

| Column | Type | Notes |
|--------|------|-------|
| `id` | text PK | |
| `expiresAt` | timestamp | |
| `token` | text (unique) | Session cookie value |
| `userId` | text FK -> user | CASCADE delete |

**`account`** - OAuth linked accounts

| Column | Type | Notes |
|--------|------|-------|

**`verification`** - email verification tokens (unused currently but required by Better Auth)

### Application Tables

**`managed_agent_session`** - one row per agent conversation

| Column | Type | Notes |
|--------|------|-------|
| `id` | text PK | `crypto.randomUUID()` |
| `userId` | text | FK to user (not enforced at DB level, checked in app) |
| `anthropicSessionId` | text (unique) | Anthropic's session ID |
| `title` | text | Default `"New chat"`, updated from first message |
| `updatedAt` | timestamptz | Bumped when new events are inserted |
| `agentId` | text | Anthropic agent ID used to create session |
| `environmentId` | text | Anthropic environment ID |
| `tailing` | boolean | `true` while a workflow is actively polling events |

**`managed_agent_event`** - persisted Anthropic session events

| Column | Type | Notes |
|--------|------|-------|
| `id` | text PK | `crypto.randomUUID()` |
| `sessionId` | text FK -> managed_agent_session | CASCADE delete |
| `anthropicEventId` | text | Anthropic's event ID |
| `type` | text | Event type (e.g. `user.message`, `agent.message`, `tool.use`) |
| `payload` | jsonb | Full event object from Anthropic |
| `processedAt` | timestamptz | From Anthropic's `processed_at` field |
| `occurredAt` | timestamptz | Best-effort timestamp for ordering |

**Unique constraint**: `(sessionId, anthropicEventId)` - enables idempotent `ON CONFLICT DO NOTHING` inserts during polling.

## Conventions

### IDs

All primary keys are `text` type using `crypto.randomUUID()`. Anthropic IDs are stored separately (`anthropicSessionId`, `anthropicEventId`) and never used as primary keys.

### Timestamps

- Better Auth tables use `timestamp` (without timezone) per Better Auth convention.
- Application tables use `timestamp with timezone` (`{ withTimezone: true }`).
- `updatedAt` on `managed_agent_session` uses `defaultNow()` and is explicitly updated when events are inserted.

### Cascading Deletes

- Better Auth: `session`, `account` cascade on user delete.
- Application: `managed_agent_event` cascades on `managed_agent_session` delete.
- The DELETE endpoint in `session/route.ts` explicitly deletes events first, then the session, as a safety measure.

### JSON Storage

Event payloads are stored as `jsonb`. The full Anthropic event object is stored, preserving all fields for future use. The `type` column is denormalized for efficient filtering.

## Migrations

```bash
# Push schema changes directly (development)
pnpm db:push

# Generate migration files (production)
pnpm db:generate
```

Drizzle config is in `drizzle.config.ts`. It loads `.env.local` via `dotenv/config` for the `DATABASE_URL`.

## Adding New Tables

1. Define the table in `lib/schema.ts` using Drizzle's `pgTable()`
2. Export it (the schema object is passed to both Drizzle and Better Auth's adapter)
3. Run `pnpm db:push` to apply changes
4. Update this doc
